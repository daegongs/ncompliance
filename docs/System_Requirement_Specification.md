# 사규관리 시스템 (nCompliance) 개발 요구사항 정의서

**문서 버전**: 1.1  
**작성일**: 2026년 1월 8일  
**최종 수정일**: 2026년 1월 8일  
**프로젝트명**: nCompliance (Corporate Regulation Management System)

---

## 1. 개요

### 1.1 목적
본 문서는 기업의 사규(정책, 규정, 지침, 매뉴얼 등)를 체계적으로 관리하고, 제·개정·폐지 절차를 표준화하며, 임직원에게 사규 열람 서비스를 제공하는 '사규관리 시스템'의 개발 요구사항을 정의합니다.

### 1.2 시스템 범위
- 사규의 등록, 조회, 수정, 삭제 및 버전 관리
- 사용자 역할 기반 접근 권한 관리
- 사규 브라우저 (트리 구조 열람)
- 알림 및 보고서 기능
- 법인/부서/사용자 조직 관리

### 1.3 용어 정의

| 용어 | 정의 |
|------|------|
| 사규 | 회사의 정책, 규정, 지침, 매뉴얼 등 모든 내부 규범 문서 |
| 제정 | 새로운 사규를 처음 만들어 시행하는 것 |
| 개정 | 기존 사규의 내용을 변경하는 것 |
| 폐지 | 기존 사규의 효력을 소멸시키는 것 |
| 책임부서 | 해당 사규의 제정/개정/폐지 및 관리를 담당하는 부서 |
| 정기검토 | 사규의 현행화를 위해 일정 주기로 내용을 검토하는 것 |

## 2. 사용자 역할 및 권한

### 2.1 역할 정의

| 역할 | 코드 | 설명 |
|------|------|------|
| 일반 사용자 | `GENERAL` | 일반 임직원. 공개된 사규 조회만 가능 |
| 책임부서 담당자 | `DEPT_MANAGER` | 소관 사규의 등록/수정/삭제 담당. 본인 부서 사규만 관리 |
| 준법지원인 | `COMPLIANCE` | 전체 사규 관리 권한. 비공개 사규 조회 가능 |

## 3. 기능 요구사항

### 3.1 사규 관리

#### 3.1.1 사규 등록 (REQ-REG-001)
| 항목 | 내용 |
|------|------|
| **설명** | 새로운 사규를 시스템에 등록 |
| **권한** | 책임부서 담당자, 준법지원인, 시스템 관리자 |
| **입력 항목** | |
| - 사규코드 | 자동 생성 (분류별 접두사 + 순번): POL0001, REG0001, GUI0001, MAN0001 |
| - 사규명 | 필수, 최대 200자 |
| - 분류 | 필수, 선택: 정책/방침, 규정, 지침, 매뉴얼/가이드라인 |
| - 그룹 | 필수, 선택: 이사회, HR, 재무, 구매, 준법, 반부패, 정보보호, 개인정보보호, 안전보건, ESG 등 |
| - 설명 | 선택, 사규에 대한 간략한 설명 |
| - 책임부서 | 필수, 부서 선택 (책임부서 담당자는 자동 고정) |
| - 사규관리 담당자 | 선택, 텍스트 입력 (책임부서 담당자는 자동 입력) |
| - 사규관리 책임자 | 선택, 텍스트 입력 |
| - 의무준수 | 선택, 체크박스 |
| - 임직원 공개 | 선택, 체크박스 |
| - 적용범위 | 선택: 전 임직원, 소속부서 |
| - 시행일 | 선택, 날짜 |
| - 정기검토예정일 | 선택, 날짜 |
| - 상위 사규 | 선택, 사규 선택 |
| - 관련 사규 | 선택, 복수 선택 |
| - 태그 | 선택, 콤마로 구분 입력 |
| - 접근 권한 | 필수, 선택: 전체 직원, 지정된 부서, 지정된 직원 |
| - 허용 법인/부서/직원 | 접근 권한에 따라 선택 |
| - 사규 원본 파일 | 선택, PDF/Word/HWP 등 (최대 10MB) |
| - 규정 본문 | 선택, 텍스트 입력 |
| - 참조 링크 | 선택, URL 입력 |
| **처리** | - 사규코드 자동 생성<br>- 상태는 '시행중'으로 설정<br>- 작성자 정보 자동 기록 |
| **비고** | 책임부서 담당자는 본인 부서로 책임부서 자동 고정 |

#### 3.1.2 사규 조회 (REQ-REG-002)
| 항목 | 내용 |
|------|------|
| **설명** | 등록된 사규 목록 조회 및 상세 정보 확인 |
| **권한** | 모든 사용자 (권한에 따른 데이터 필터링) |
| **목록 화면** | |
| - 검색 조건 | 검색어(사규명, 코드, 담당자, 그룹), 분류, 상태, 의무준수, 책임부서, 임직원 공개 |
| - 표시 항목 | 사규코드, 그룹, 분류, 사규명, 책임부서, 담당자, 버전, 상태, 의무준수 |
| - 페이징 | 20건 단위 |
| **상세 화면** | |
| - 기본 정보 | 모든 사규 속성 표시 |
| - 버전 이력 | 제정/개정/폐지 이력 목록 |
| - 관련 사규 | 상위/하위/관련 사규 링크 |
| - 태그 | 태그 목록 및 링크 |
| **권한별 필터링** | |
| - 일반 사용자 | 공개 사규만 조회 |
| - 책임부서 담당자 | 본인 부서 담당 사규만 조회 |
| - 준법지원인/관리자 | 전체 사규 조회 |

#### 3.1.3 사규 수정 (REQ-REG-003)
| 항목 | 내용 |
|------|------|
| **설명** | 등록된 사규의 정보 수정 |
| **권한** | 책임부서 담당자(소관만), 준법지원인, 시스템 관리자 |
| **제약조건** | - 사규코드는 수정 불가<br>- 책임부서 담당자는 책임부서 변경 불가 |

#### 3.1.4 사규 삭제 (REQ-REG-004)
| 항목 | 내용 |
|------|------|
| **설명** | 등록된 사규 삭제 |
| **권한** | 준법지원인, 시스템 관리자 |
| **처리** | - 삭제 확인 메시지 표시<br>- 관련 버전, 태그 연결, 즐겨찾기 등 연쇄 삭제 |
