# 사규관리시스템(nCompliance) 개발 요구사항 정의서

## 1. 개요

본 문서는 사규의 제정, 개정, 폐지 및 상시 조회를 체계적으로 관리하기 위한 '사규관리시스템'의 개발 요구사항을 정의합니다.

> [!NOTE]
> 사규의 실제 결재 승인 프로세스는 사내 표준 전자결재 시스템 등 별도 시스템에서 진행되며, 본 시스템은 결재 결과 연동 및 사규 현행화 관리에 집중합니다.

## 2. 사용자(일반 임직원) 요구사항

일반 사용자는 회사의 규정을 쉽고 빠르게 검색하고 열람할 수 있어야 합니다. 단, 자신이 권한을 보유한 규정만 볼 수 있습니다.

| ID | 요구사항명 | 상세 내용 |
| :--- | :--- | :--- |
| U-00 | 초기 화면(대시보드) | **초기 화면 진입 시 전체 규정을 카테고리별로 브라우징할 수 있는 카드/메뉴 형태로 구성 (권한 있는 규정만 노출)** |
| U-01 | 사규 통합 검색 | 제목, 본문 키워드, 사규 번호 등을 통한 강력한 검색 기능 제공 |
| U-02 | 카테고리별 전용 페이지 | **정책/방침, 규정, 지침, 매뉴얼/가이드라인별 전용 브라우징 페이지 제공 및 필터링** |
| U-03 | 권한 기반 규정 노출 | **모든 브라우징 및 검색 결과에서 사용자가 접근 권한을 가진 규정만 노출되도록 강제** |
| U-04 | 사규 상세 내역 확인 | 사규의 목적, 시행일, 최신 버전, 담당 부서 정보 확인 |
| U-05 | 첨부파일 다운로드 | 사규 원문(PDF, Docx 등) 다운로드 및 다운로드 시 워터마크 적용(선택) |
| U-06 | 제·개정 알림 수신 | 사내 공통 사규의 변경 또는 신규 등록 시 팝업/이메일 알림 수신 |
| U-07 | 개인 북마크 | 자주 찾는 사규를 '즐겨찾기'에 등록하여 퀵 메뉴에서 접근 |
| U-08 | 관련 사규 연결 조회 | 현재 보고 있는 사규와 연관된 상위/하위/참조 사규로 즉시 이동 |

## 3. 규정책임부서 담당자 요구사항

사규를 운영하는 부서의 담당자는 사규의 생애주기(Life-cycle)를 관리하며, 별도 결재 시스템과의 연동 정보를 처리합니다.

| ID | 요구사항명 | 상세 내용 |
| :--- | :--- | :--- |
| M-01 | 사규 제·개정 정보 등록 | 제·개정된 사규의 최종본 파일 업로드 및 관련 메타데이터 입력 |
| M-02 | 버전 관리(History) | 버전별 변경 사유, 변경 내용 요약 기록 및 이전 버전과의 비교 기능 |
| M-03 | 외부 결재 정보 연동 | 별도 결재 시스템에서 승인된 결재 번호/일자 연동 및 승인 문서 링크 저장 |
| M-04 | 접근 권한 설정 | 사규별 접근 범위 설정 (전체/특정 법인/특정 부서/특정 개인) |
| M-05 | 정기 검토 알림 관리 | 사규별 정기 검토 주기를 설정하고, 기한 도래 시 담당자에게 알림 발송 |
| M-06 | 폐지 프로세스 | 사규의 효력 상실 시 폐지 사유 등록 및 상태 변경(시행중 -> 폐지) |
| M-07 | 태그 및 메타데이터 관리 | 검색 편의를 위한 키워드(태그) 지정 및 책임부서 정보 현행화 |
| M-08 | 관련 규정 매핑 | 사규 간의 상속 관계(상위-하위) 및 참조 관계 설정 |

## 4. 관리자(시스템 관리자) 요구사항

시스템 관리자는 조직 정보 및 전반적인 시스템 설정을 관리합니다.

| ID | 요구사항명 | 상세 내용 |
| :--- | :--- | :--- |
| A-01 | 조직/사용자 관리 | 법인(Company), 부서(Department), 사용자 정보 동기화 및 수동 관리 |
| A-02 | 권한 그룹 관리 | 관리자, 준법지원인, 부서 담당자 등 역할별 시스템 접근 권한 제어 |
| A-03 | 인터페이스 관리 | 외부 결재 시스템 및 HR 시스템과의 API 연동 상태 모니터링 |
| A-04 | 감사 추적(Audit Log) | 사규 조회 및 다운로드 이력(IP, 시간, 사용자) 로깅 및 리포트 |
| A-05 | 시스템 대시보드 | 전사 사규 현황(상태별, 부서별 통계) 시각화 및 리포팅 기능 |
| A-06 | 게시판 및 공지 관리 | 시스템 공지사항 및 Q&A 게시판 운영 관리 |
| A-07 | 마스터 데이터 관리 | 사규 분류 코드, 법인 코드 등 시스템 표준 코드 관리 |

## 5. 초기 화면 및 카테고리 페이지 구성 (상세)

각 권한별 초기 화면 및 카테고리 페이지는 다음과 같이 구성합니다.

### 5.1 초기 화면 브라우징 구성

- **카테고리 맵**: 화면 상단 또는 중앙에 4대 카테고리(정책/방침, 규정, 지침, 매뉴얼)를 한눈에 볼 수 있도록 배치
- **동적 카운팅**: 각 카테고리 아래에는 사용자가 접근 가능한 사규의 개수 표시

### 5.2 카테고리별 전용 페이지

다음 페이지들을 각각 독립적으로 추가하여 브라우징을 지원합니다.

1. **정책/방침 페이지**: 회사의 최상위 원칙 및 가이드라인 목록
2. **규정 페이지**: 업무 수행의 기본 틀이 되는 제규정 목록
3. **지침 페이지**: 규정의 세부 실행을 위한 부서별/업무별 지침 목록
4. **매뉴얼/가이드라인 페이지**: 실무 참고용 매뉴얼 및 가이드 목록

### 5.3 브라우징 원칙

- **접근 제어 필터링**: 모든 페이지 로딩 시 현재 로그인한 사용자의 법인, 부서, 개인별 접근 권한(ACL)을 체크하여 권한이 있는 규정만 렌더링
- **역할별 차별화**:
  - 일반 사용자: 본인이 읽을 수 있는 시행 중인 사규 중심
  - 책임부서 담당자: 본인 소관 사규의 초안, 검토중 상태를 포함하여 브라우징 가능

## 6. 비기능 및 공통 요구사항

- **보안**: 중요 사규 파일 암호화 저장 및 권한 없는 사용자의 URL 직접 접근 차단
- **UI/UX**: 직관적인 UI(Modern Web Design), 반응형 웹 지원(모바일 열람 고려)
- **성능**: 대량의 사규 데이터 조회 시 페이징 및 인덱싱을 통한 최적화
- **연동(중요)**: 별도 전자결재 시스템의 승인 결과 수신을 위한 Webhook 또는 API 인터페이스 구축
